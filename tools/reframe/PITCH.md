# Reframe — Pitch

> **A live thinking canvas. Talk, and the wall rebuilds itself — and never loses an idea.**

## The problem

Brainstorms, planning, and structuring sessions still run on sticky notes. We cover a wall,
then someone photographs it and **re-types everything** afterwards — losing ideas, momentum,
and an hour of everyone's evening. Worse, the act of *formalizing* (writing it down, drawing the
diagram) interrupts the thinking. You stop having ideas in order to record the last one.

## The solution: Reframe

Reframe removes that friction. You just talk. A [Bee](https://bee.computer/) wearable listens;
spoken ideas are distilled into nodes, **auto-clustered into themes**, and projected on a live
canvas the whole room can see. Comment on what you want — *"group these by priority"*, *"show
this as a mindmap"* — and the canvas **re-shapes itself in real time**. Nothing is ever lost: the
model is append-only, and a coverage badge proves it — **"N/N ideas captured · 0 lost."**

The communication channel is the projected surface itself. You point and speak; the agent reads
intent and re-authors the visual. Structure stops being homework and becomes a live by-product of
the conversation.

## Why this fits the workshop — the two primitives, reframed

Reframe is built directly on the workshop's two perception primitives, repurposed for a new domain:

| Workshop primitive | Reframe reframing |
|---|---|
| **Annotation** — perception of clicks on a rendered DOM | Perception of **speech** via Bee — you talk instead of clicking |
| **Verification** — every CSS rule checked against an expected value | **Coverage** verification — every spoken utterance must map to a node (0 lost) |

The same philosophy — a shared perceptual surface plus deterministic verification — applied to
**thinking** instead of CSS.

## How it works (90-second architecture)

```
Bee voice ──▶ transcript ──▶ Reframe server ──▶ AI brain (Claude) ──▶ JSON patch
                                  │  append-only state (nodes · clusters · view · coverage)
                                  ▼  live SSE
                          Canvas: board · list · mindmap  +  coverage badge  +  one-click export
```

- **Append-only model** — ideas are captured forever; re-clustering and view changes never delete
  anything. "Never lose an idea" is structural, not a feature that can regress.
- **AI brain returns data, not prose** — Claude emits a strict JSON patch (new nodes, clusters,
  view); the server validates and applies it. If the model ever fails, a fallback still captures
  the raw idea, so the canvas can't drop input.
- **Coverage = verification** — after every change, Reframe checks that each idea-bearing
  utterance produced a node. The badge is the audience-visible proof.
- **Zero-dependency surface** — the canvas is a single self-contained file; runs on any laptop,
  can't fail on stage.

## Live demo flow

1. Empty canvas on the projector. The team starts brainstorming out loud.
2. Sticky notes appear and **auto-cluster** into themes as people speak.
3. Someone says **"Can we see this as a mindmap?"** → the canvas morphs into a radial mindmap.
4. Someone says **"Group these by priority."** → the clusters reorganize.
5. End on **"N/N ideas captured · 0 lost"** → one click exports a shareable markdown of the session.

## Where it goes next: Generative UI

Today the AI brain *picks* from a fixed palette of views (board, list, mindmap). The ceiling is the
set of views we pre-coded. The next step removes that ceiling with **generative UI** (via
[CopilotKit](https://copilotkit.ai)): the agent no longer *chooses* a view — it **composes** one.

> *"Show these ideas as an impact/effort 2×2."* *"Make it a RICE table."* *"Draw the dependency
> graph."* — and the surface renders a layout we never hand-coded.

This is the literal realization of the premise: **say you need a new view, and it appears — any
view, generated on the fly.** CopilotKit provides the three building blocks:

- **Generative UI** (`useCopilotAction` with a `render` function) — the agent streams a React
  component into the surface; Reframe exposes composable primitives (grid, axes, swimlane,
  node-graph) the agent assembles.
- **CoAgents / shared state** — the append-only canvas becomes the agent's shared state; typed chat
  and Bee voice drive one canvas.
- **Human-in-the-loop + frontend actions** — *"merge these two"*, *"move that to Risks"* — the agent
  acts on the canvas and the human approves.

The current zero-dependency canvas stays as the de-risked demo; generative UI is the v2 once the
concept is proven live. Full plan: [ROADMAP.md](ROADMAP.md).

## The one-liner

**Reframe turns a conversation into a structure — live, hands-free, and provably complete — and is
on a path where the interface itself is generated by the agent on demand.**
